## Usage Rule

Public product messaging should be driven by the root README and the user-facing docs in `../`.

Internal notes can describe historical tradeoffs, reference implementations, and unfinished experiments, but they should not replace the public product definition.

When updating internal notes, preserve prior conclusions, background, architecture direction, and dated milestones. If a decision changes, append the new conclusion with date and rationale instead of deleting historical context.
